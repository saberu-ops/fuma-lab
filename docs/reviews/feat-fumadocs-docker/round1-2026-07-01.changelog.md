### Project Governance

- `docs/reviews/feat-fumadocs-docker/`: contains the approved plan and this
  implementation record.

## Intent

- Keep the host clean by placing Node.js and application dependencies inside
  the image.
- Keep runtime state disposable by compiling MDX into the image.
- Preserve server-side features through Next.js standalone output.
- Avoid accidental public exposure until authentication and a reverse proxy
  are explicitly selected.
- Make public metadata configurable at build time through `SITE_URL`.

## Explicitly Not Done

- Authentication, authorization, TLS, and public reverse proxy.
- AI chat or external model provider integration.
- CMS or collaborative browser editor.
- Mandarin-specific Orama tokenizer configuration.
- Automated image publishing or deployment.

passed.

## Known Dependency Finding

`npm audit` reports two moderate findings for PostCSS 8.4.31 nested in the
current stable Next.js 16.2.9 package. On 2026-07-01, Next.js 16.2.9 is the
latest stable version and npm offers only a breaking downgrade to Next.js 9 as
an automatic fix. No override or forced downgrade was applied. Recheck after a
patched stable Next.js release.
