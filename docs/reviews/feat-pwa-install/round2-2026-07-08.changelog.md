## What went wrong (Round 1 defect)

Round 1 branched `feat/pwa-install` from `main` (e8e3314). But two commits
lived only on `docs/design-artifact`, never merged to `main`:

- `ba7a632` docs: add project design overview artifact
- `5fdccc9` feat: add JLPT N2 exam archive workspace

They were the actually-deployed state (the previous container, built 07-06,
served them). Building/deploying from a `main`-based branch dropped them.
Post-deploy the live site regressed: `/docs/japanese-n2/past-exams` and
`/docs/japanese-n2/past-exams/2024-07/listening` returned **404**, the
`listening-to-speaking → past-exams` redirect vanished, and the
`listening-practice.tsx` rewrite + design docs were gone (26 files, +1447).

Root cause: base-branch selection did not account for unmerged work on the
branch the session started on. The deploy smoke test only asserts generic
markers (`/docs`, "个人文档", audio range), so it did not catch the loss.

## Fix

- `git merge --no-ff docs/design-artifact` into `feat/pwa-install`
  (merge commit `d9474a7`). No conflicts — the two branches touch disjoint
  files (PWA assets vs. content/player). Non-destructive: no history rewrite,
  no force-push.
- Rebuilt and redeployed the combined tree.
