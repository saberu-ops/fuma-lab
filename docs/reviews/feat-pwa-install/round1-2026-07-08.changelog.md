## Intent

Make the site installable as a PWA so browsers show the address-bar
"Install" action and it can be launched as a standalone app. Uses Next.js
16 native primitives only — no `next-pwa` or other third-party wrapper
(W-R18: `app/manifest.ts` + `public/sw.js` is the framework-recommended path).

## File-change manifest

| File | Change | Purpose |
|------|--------|---------|
| `app/manifest.ts` | new | Web App Manifest via `MetadataRoute.Manifest`. Next serves `/manifest.webmanifest` and auto-injects `<link rel="manifest">`. Declares name/short_name/description, `start_url:/`, `scope:/`, `display:standalone`, background/theme color, and 3 icons (192 any, 512 any, 512 maskable). |
| `scripts/generate-pwa-icons.mjs` | new | Regeneratable placeholder-icon generator (sharp). Emits `public/icons/{icon-192,icon-512,icon-maskable-512,apple-touch-icon,favicon-32}.png`. Dark rounded square + "FL" monogram; maskable variant padded into the safe zone. |
| `public/icons/*.png` | new | Generated icon assets (committed). Replace with real brand art later, keeping filenames. |
| `public/sw.js` | new | Minimal service worker: `install`/`activate`/`fetch`. Pass-through fetch handler (no caching) — satisfies the Chromium installability requirement without serving stale docs. |
| `components/pwa-register.tsx` | new | `'use client'` component registering `/sw.js`. Skips non-production and non-secure-context; registers on `load`. Renders nothing. |
| `app/layout.tsx` | edit | Add `viewport.themeColor` (light/dark), `metadata.appleWebApp`, `metadata.icons.apple`, and render `<PwaRegister />`. |

## Explicitly NOT done

- **No offline / asset caching.** Deliberate — frequently-updated docs site;
  a caching strategy is a separate future round if wanted.
- **No TLS/reverse-proxy changes.** HTTPS is provided externally via
  Cloudflare Tunnel (confirmed by operator); `compose.yaml` unchanged.
- **No real brand icons.** Placeholder monogram icons only.
- No commit/push performed (awaiting operator).
