# Round 1 Changelog: Deterministic Codex Deployment

## Scope

Add a repository-owned deployment workflow so Codex and operators can validate,
release, verify, and roll back the local documentation service through one
reviewed command.

## File-Change Manifest

- `scripts/deploy.sh`: adds the deterministic deployment entry point with
  prerequisite validation, a cross-process lock, running-image preservation,
  Docker build, health polling, runtime-isolation checks, application smoke
  tests, and automatic rollback.
- `package.json`: adds `verify`, `deploy`, and `deploy:check`.
- `Dockerfile`: runs lint, snapshot verification, and type checking before the
  production build.
- `AGENTS.md`: makes deployment the default completion step after
  application-visible changes while defining explicit opt-out and failure
  boundaries.
- `.agents/skills/deploy-fuma-lab/`: adds the repository-scoped Codex Skill and
  UI metadata.
- `.codex/rules/deploy.rules`: allows only the reviewed `npm run deploy`
  command prefix to leave the workspace sandbox.
- `.gitignore`: excludes Python bytecode and `__pycache__` directories produced
  by subtitle tooling.
- `README.md` and `docs/features/fumadocs/operations.md`: document the operator
  and Codex workflows.

## Operational Behavior

`npm run deploy` performs the following sequence:

1. Acquire an exclusive deployment lock.
2. Validate the Compose configuration and Git whitespace across staged and
   unstaged tracked changes.
3. Preserve the running image under a timestamped rollback tag.
4. Build the new image; the Docker build runs `npm run verify` and
   `npm run build`.
5. Replace only the `docs` service and wait for its health check.
6. Confirm the new image ID, non-root/read-only isolation, dropped
   capabilities, `no-new-privileges`, and PID limit.
7. Smoke-test the docs root, Japanese N2 page, search, and audio byte ranges.
8. Restore and verify the previous image automatically if a post-switch step
   fails.

`npm run deploy:check` uses a timestamped candidate image tag through a
temporary Compose override. It does not change `fuma-lab:local` or replace the
running container.

## Codex Activation

Codex reads `AGENTS.md`, repository Skills, and project rules when it starts a
new session. The repository must be trusted for project-local `.codex` rules to
load. Operators can invoke `$deploy-fuma-lab` explicitly; matching content and
deployment tasks can invoke it implicitly.

## Reproducibility Note

The deployment was built from the reviewed dirty working tree. The deployment
cannot be reproduced from Git history until the current changes are committed.
