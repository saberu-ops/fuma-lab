## Intent

Add a staging environment on `stg.t3s7.com` — same app as prod, same host, fully
isolated — via a containerized Cloudflare Tunnel, gated by Cloudflare Access and
`noindex`. Safe-by-default: a bare `docker compose` is a harmless sandbox, never
production (operator decision, folded in during the round).

## Independent review (destructive-sink, §3)

`deploy.sh` is destructive-sink; per §3 it got review beyond self-review. Two
defects were found and fixed **before** any prod action:
1. **deploy.sh ordering** — `IMAGE`/`BASE_URL` were derived before the pin block,
   so a custom `envs/production.env` port/image would desync the smoke-test URL /
   rollback. Fixed by pinning first, then deriving.
2. **staging.sh smoke pipe race** — `curl | grep -q` under `pipefail` could
   SIGPIPE curl on match and report a false "noindex missing". Fixed by capturing
   headers, then grep.

> Review method note (waiver): the two planned independent subagent reviews were
> repeatedly killed by SSH drops; the review was completed inline plus empirical
> `docker compose config` resolution checks. Re-running `/code-review` or the
> subagents when the link is stable is advisable before merge.
