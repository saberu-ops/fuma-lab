## Intent

- Reproduce the official Layout Tabs dropdown with native Fumadocs page-tree
  roots.
- Keep frequently used Fumadocs guidance available locally and searchable.
- Preserve a small dependency set by consuming processed Markdown instead of
  the upstream monorepo's MDX build stack.
- Make third-party ownership, license, version, and modifications explicit.
- Keep normal application build and runtime independent of the official
  documentation site.

## Explicitly Not Done

- Full 156-page official documentation mirror.
- Automatic translation of either personal or upstream content.
- Multi-language routes or a language picker.
- Mandarin-specific search tokenization.
- OpenAPI, AsyncAPI, Twoslash, TypeScript Docgen, official AI chat, branding,
  fonts, or official-site-only interactive components.
- Scheduled or automatic upstream refresh.

## Deviations from Plan

- The plan expected the smoke-test container to be removed. A user-started
  service was already running, so it was replaced with the verified image and
  intentionally left healthy at `127.0.0.1:3000`.
- The official `/docs` root did not honor Markdown content negotiation, while
  `/docs/index` returned the same page as validated Markdown. The fixed
  allowlist therefore uses `/docs/index` only as the fetch route and retains
  `/docs` as the recorded source and local mapping.
