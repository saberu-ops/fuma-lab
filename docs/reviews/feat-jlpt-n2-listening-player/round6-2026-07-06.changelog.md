# Round 6 Changelog: Scroll-Free Interactive Page Sizing

## Scope

Keep the complete listening workspace inside the desktop viewport with either
sidebar state, while allowing the player to grow horizontally when the sidebar
is collapsed.

## Changes

- Moved desktop height ownership from a viewport-offset estimate on the player
  to the interactive `DocsPage` container.
- Let the title take its natural height and the player consume the exact
  remaining flex space.
- Disabled document breadcrumbs and footer navigation on interactive pages so
  they cannot add hidden vertical overflow.
- Removed the standard 1168px page-width cap for interactive pages; the player
  now follows the main column width as the sidebar opens and closes.
- Kept the page container overflow hidden on desktop while retaining internal
  scrolling for the question strips and transcript.
- Added missing `min-width: 0` and `min-height: 0` constraints so both player
  columns can shrink without forcing page overflow.
- Preserved the natural, scrollable stacked layout below the desktop
  breakpoint.

- the deployed response contains the docs-layout height constraint, uncapped
