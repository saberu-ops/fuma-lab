# Round 1 Changelog: JLPT N2 Listening Player

## Plan Reference

No separate plan file. Implemented directly from the request to split the
provided July 2024 JLPT N2 recording and publish it as an interactive listening
exercise in the existing Fumadocs Docker instance.

## File-Change Manifest

- `components/listening-practice.tsx`: adds the client-side listening player,
  section and question navigation, synchronized transcript, cue seeking,
  blind-listening mode, playback speed, sentence repeat, and session progress.
- `components/mdx.tsx`: exposes the listening player to MDX pages.
- `content/docs/(personal)/japanese-n2/listening-to-speaking/2024-july-listening-exam.mdx`:
  adds the full-width listening exercise and recommended practice workflow.
- `content/docs/(personal)/japanese-n2/listening-to-speaking/index.mdx`: links
  the new exercise from the existing listening topic.
- `scripts/build-listening-exam.mjs`: parses the supplied timing document,
  repairs malformed cue timings for playback, emits the data manifest, and
  splits the source recording with FFmpeg.
- `lib/data/jlpt-n2-2024-07.json`: stores 5 sections, 29 clips, and 222
  synchronized transcript cues.
- `public/audio/jlpt-n2/2024-07/*.mp3`: contains 29 browser-ready mono MP3
  clips at 80 kbit/s.
- `package.json`: adds the `audio:build` command.

The supplied authoring inputs remain at the repository root as
`202407.md`, `202407.mp3`, and `202407.txt`.

## Operational Decisions

- Audio boundaries follow each `文件名` range in `202407.md`.
- Eight malformed source cue timings are repaired only in the generated
  playback manifest; the supplied source timing document is preserved.
- Generated web audio totals 22,330,748 bytes and 2,232.19 seconds.
- The deployment intentionally uses the reviewed dirty working tree. Until
  these files are committed, the running image cannot be reproduced from Git
  history alone.
- The existing loopback-only Compose network and container isolation settings
  were preserved.

## Known Limitations

- Subtitle text and source timing quality are limited by the supplied
  transcript. The generated manifest repairs timing order but does not rewrite
  Japanese transcription errors.
- Completion state is session-local React state and is not persisted across
  page reloads.
- The service remains available only on `127.0.0.1:3000`; no public exposure,
  TLS, or authentication was added.
