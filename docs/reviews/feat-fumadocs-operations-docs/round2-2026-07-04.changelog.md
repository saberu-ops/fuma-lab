  Chinese/Japanese/English search behavior instead of the previous
  English-only baseline.
- Distinguish a reproducible clean-tree release from an explicitly reviewed
  build of current uncommitted work.
- Keep dependency-audit guidance correct when npm clients and the advisory
  database report different counts for the same lock file.
- Start the actual Compose workload through the documented build-then-switch
  workflow and verify its application and isolation behavior.
- Preserve the loopback-only, non-root, read-only deployment boundary.

## Dependency Observation

The container build used npm 11.16.0 and reported two moderate advisories plus
unapproved install-script notices for `esbuild` and `sharp`. The host npm 9
installation had reported five moderate advisories for the same locked project.
This confirms that the runbook must require a fresh audit and dependency-path
review instead of treating a historical count as permanent.

Both the host and container production builds completed without approving the
pending install scripts. No forced audit fix was applied.

## Known Limitations

- The running image contains the current uncommitted working tree and cannot be
  reproduced from Git history alone until the intended files are committed.
- `SITE_URL` uses the documented default `http://localhost:3000`; changing the
  public hostname requires a rebuild.
- The service has no authentication or TLS and must remain loopback-only until
  an external access design is selected.
- CPU and memory limits remain unset pending real workload observations.
- CJK search uses deterministic bigrams rather than morphological analysis, and
  single-character CJK queries are not its target use case.
