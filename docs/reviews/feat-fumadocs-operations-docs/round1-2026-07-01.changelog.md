## Intent

- Keep README short enough for first use while making the default Docker path
  complete.
- Give operators exact commands that match the current service, image,
  variables, routes, health check, and immutable-content model.
- Make the build-time behavior of `SITE_URL` and runtime port behavior of
  `DOCS_PORT` explicit.
- Provide safe release and rollback procedures for the mutable local image
  tag.
- Treat Git and source content as the durable state because the service has no
  database or persistent volume.
- Keep the lack of authentication, TLS, and public reverse proxy visible at
  both onboarding and operations levels.

## Operational Decisions

- Routine commands target the Compose service `docs`, not its generated
  container name.
- Image builds and container switching are documented as separate steps so a
  failed build does not replace the running service.
- Operators preserve the current image under a timestamped rollback tag before
  rebuilding `fumadocs-personal:local`.
- Git bundle and image archive examples write outside the repository to avoid
  accidental commits.
- Snapshot refresh remains an explicit networked supply-chain operation;
  normal builds and runtime remain offline with respect to the official docs.
- The runbook documents the current dependency finding without recommending
  a breaking `npm audit fix --force`.

## Explicitly Not Done

- No application, dependency, Docker, Compose, snapshot, route, search, or

- No authentication, TLS, reverse proxy, monitoring, CI/CD, scheduled backup,
  or image registry was added.
- No public deployment was performed.
- The existing healthy container and local image were not replaced.

## Known Limitations

- Authentication, TLS, and public reverse proxy remain external prerequisites.
- Search tokenization remains English, so Chinese search quality is limited.
- Rollback tags remain local unless an image archive or registry is added.
- The repository currently has no configured remote; a local commit alone is
  not an off-host backup.
- The 2026-07-01 dependency baseline retains two moderate findings from
  PostCSS 8.4.31 nested in Next.js 16.2.9.
