§ 01

四本设计支柱

四个关注点相互正交：内容体系、离线参考、同源检索、封存部署。任意一个都可独立演进而不牵动其余。

Content · 内容

双根内容体系

搜索框下方两个 Fumadocs 原生 Layout Tab，分隔个人与参考两类内容。

  • (personal) 中文个人文档，路由组括号不进 URL，入口仍从 /docs 起
  • 日语 N2 学习区：核心词汇语法 + 听力转口语精读
  • fumadocs 官方英文参考快照，入口 /docs/fumadocs
Reference · 参考

可校验离线快照

23 篇精选官方文档随镜像构建，正常构建与运行不访问官网。

  • 固定白名单 + SHA-256 清单锁定 third_party/fumadocs/snapshot.json
  • docs:sync 是唯一联网动作，更新后需人工审阅差异
  • docs:check 校验文件、白名单与哈希一致
Search · 检索

中日英混合分词

本地 mixed-cjk 分词器补足 Orama 内置分词器不切分 CJK 的缺口。

  • CJK 文本索引为重叠二元组，单字则为一元组
  • 拉丁词整体保留；先做 NFKC 归一化并小写
  • 兼顾中文、日文语块与英文技术词，不过度放宽
Deploy · 部署

封存式容器发布

非 root 标准镜像，只读根文件系统，端口仅绑定宿主回环地址。

  • 单一入口 npm run deploy 完成校验、发布、验证、回滚
  • 切换后自检失败自动恢复上一镜像
  • Codex 通过 $deploy-fuma-lab Skill 复用同一入口

§ 03

内容分类体系

个人根以「专题 → 课次」组织，可扩展；参考根是固定的官方文档子集。路由组 (personal) 只做组织、不出现在地址中。

个人根 · japanese-n2

围绕 JLPT N2 备考组织。核心词汇语法区把 32 个核心词与 13 个语法句型拆到六课；听力转口语区把真题音频作精读，配一个课内听力练习组件。

参考根 · fumadocs

官方文档的 23 篇精选英文快照，覆盖 UI、MDX、搜索、集成、部署与国际化，供离线查阅框架用法。

扩展方式

新增一篇听力转口语笔记走 npm run note:new 脚手架，保证目录约定与页面结构一致；新专题遵循同一「专题 / meta.json / 课次」骨架。

§ 05

听力真题数据管线

把两份不可直接使用的字幕源——保留真实时间轴的粗 ASR 与措辞更好但时间轴另属一版的转写——对齐成可用的整句字幕，再切成练习片段。

校准 · calibrate-listening-subtitles.py

套用审阅措辞，把短语拼成整句与应答；保留题号与选项为语义单元；仅在日文逗号停顿处按 44 字上限切分长句；以 0.967 相似度对齐两条文本流，用 439 个可靠文本/时间锚点与 1,121 个检测到的停顿，把边界映射回真实录音。

成片 · build-listening-exam.mjs

直接解析并校验标准 SRT；从 342 条源 cue 生成数据与音频，配置边界外的 cue 被排除。

§ 06

确定性部署管线

npm run deploy（即 scripts/deploy.sh）的八段顺序——每一步是下一步的前提，任一后置检查失败即自动回滚。此处编号即真实执行次序。

1

取得独占部署锁

跨进程锁，避免并发发布互相踩踏。

2

校验 Compose 与 Git 空白

检查 Compose 配置，以及已暂存/未暂存跟踪改动的空白规范。

3

封存当前镜像为回滚点

把正在运行的镜像打上带时间戳的回滚标签。

4

构建新镜像

Docker 构建内部运行 npm run verify 与 npm run build。

5

只替换 docs 服务

切换容器并等待其健康检查通过。

6

确认运行时隔离

核对镜像 ID、非 root/只读、已丢弃能力、no-new-privileges 与 PID 上限。

7

冒烟测试

验证文档根、日语 N2 页、搜索与音频字节范围。

8

失败自动回滚

任一切换后步骤失败，自动恢复并验证上一镜像。

§ 07

安全边界（当前基线）

基线刻意最小：只在受信宿主本机可达，把认证与传输安全留给受控代理层。这是一条明确的设计约束，不是遗漏。

基线已保证

  • 容器非 root 运行，根文件系统只读
  • 丢弃 Linux 能力，no-new-privileges，限制 PID
  • 仅发布 127.0.0.1:${DOCS_PORT}:3000
  • 凭据永不入库，构建与运行离线

基线不含 · 需上层补齐

  • 身份认证与访问控制
  • TLS 与公网反向代理
  • 不得改绑 0.0.0.0 或直接暴露公网
  • 远程访问须先在受控代理/私网实现 TLS 与鉴权
